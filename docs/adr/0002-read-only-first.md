# ADR-0002: 初期スコープを read-only に限定する

## Status

Accepted

## Context

Dalamud の公開ガイドラインでは、自動化、戦闘介入、PvP 優位、過度な解析、他人の識別情報収集は強く制限される。MCP は LLM と外部ツールをつなぐ性質上、機能境界を曖昧にすると意図せず自動化寄りの設計になりやすい。

## Decision

初期版では、外部に公開する能力を read-only の文脈取得に限定する。

含む:

- 自キャラ文脈
- パーティ概況
- ターゲット概況
- duty / zone / UI の要約
- 所持品の集約済み情報

含まない:

- 入力代行
- UI 操作代行
- 行動提案のための高頻度戦闘 telemetry
- 他者固有識別子の保存

## Consequences

良い点:

- リスク境界が明確
- 実装順序を整理しやすい
- 将来審査を受ける場合にも説明しやすい

悪い点:

- 「MCP でゲームを操作したい」という期待には応えない
- ユースケースの派手さは下がる
